# Livebild / Webcam

Dieses Dokument behandelt die optionale Integration eines Livebilds oder einer Webcam in die Zuschaueranzeige.

## Status

Die Webcam-Integration ist für Version 1 nicht vorgesehen. Sie kann später ergänzt werden, ohne die bestehende Anzeigestruktur grundlegend zu ändern.

Alle technischen und gestalterischen Detailfragen werden entschieden, wenn die Funktion konkret umgesetzt wird.

## Mögliche Einsatzszenarien

- Livebild der Spielfläche als Einblendung auf der Zuschaueranzeige während eines Versuchs
- Bild-in-Bild auf dem Matchday-Screen
- eigene Webcam-Ansicht, die die Turnierleitung manuell aktivieren kann

## Bereits festgelegte Punkte

| Punkt | Entscheidung |
|-------|-------------|
| Ton | Nicht vorgesehen |
| Datenschutz / Einverständnis | Wird entschieden, wenn die Webcam-Funktion umgesetzt wird |
| Alle weiteren technischen Fragen | Werden entschieden, wenn die Webcam-Funktion umgesetzt wird |

## Technische Fragen (offen bis zur Umsetzung)

Folgende Punkte sind noch nicht entschieden und werden bei der konkreten Umsetzung geklärt:

- USB-Webcam oder andere Quelle (z. B. HDMI-Capture)?
- Wird die Kamera dauerhaft verbunden sein oder nur bei Bedarf angesteckt?
- Soll die Turnierleitung die Kamera manuell aktivieren und deaktivieren können?
- Gibt es wahrnehmbare Verzögerungen im Bild?
- Muss das Kamerabild gespiegelt oder zugeschnitten werden?
- Wo im Layout wird das Kamerabild positioniert (Bild-in-Bild, eigener Bereich)?
- Wann genau wird die Webcam automatisch oder manuell eingeblendet?

## Einordnung für Version 1

Die Anzeigestruktur soll so aufgebaut sein, dass ein Webcam-Bereich später ergänzt werden kann, ohne bestehende Screens neu aufzubauen.

Für Version 1 ist kein Webcam-Code und keine Webcam-Konfiguration notwendig.
